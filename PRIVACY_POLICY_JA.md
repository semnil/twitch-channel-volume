# プライバシーポリシー — Twitch Channel Volume

最終更新日: 2026-06-06

## 概要

Twitch Channel Volume は、Twitch のチャンネルごとに音量（ゲイン）を記憶・自動適用する Chrome 拡張機能です。再生中の音声を ITU-R BS.1770 規格でラウドネス計測し、ターゲット LUFS に基づいて音量を自動調整します。本ポリシーでは、拡張機能が扱うデータの種類、利用目的、保存先について説明します。

## 収集するデータと目的

### チャンネル音量設定

- **対象**: Twitch チャンネル識別子（ライブの login 名、VOD / クリップの owner ID / broadcaster ID）、表示名、ページ URL、種別（Live / VOD / Clip）ごとに設定したゲイン値、および直近のラウドネス計測値のキャッシュ
- **目的**: 保存済みチャンネルの配信・動画・クリップを開いた時に、設定した音量を自動適用
- **保存先**: デバイス上の `chrome.storage.local` にローカル保存。外部サーバーには一切送信されません

### 拡張機能の設定

- **対象**: ターゲット LUFS 値、CM ゲイン (dB)、表示単位（% または dB）、ゲイン表示のオン・オフ
- **目的**: ユーザーの好みに応じた動作のカスタマイズ
- **保存先**: デバイス上の `chrome.storage.local` にローカル保存

### 再生音声のラウドネス計測（メモリ内のみ）

- **対象**: 再生中の `<video>` 要素の音声から Web Audio API で算出する Momentary / Short-term / Integrated LUFS 値
- **目的**: ラウドネス情報の表示および推奨ゲインの算出
- **保存**: 音声波形やオーディオデータは保存しません。計測中の LUFS 値はページを開いている間のみメモリに保持し、「チャンネルに適用」を実行した時のみ直近値が設定としてローカル保存されます

### Twitch ページ情報（読み取りのみ）

- **対象**: Twitch 自身が発行する HLS マニフェスト（CM 区間 `EXT-X-DATERANGE` の検出用）および GraphQL レスポンス（チャンネルの owner ID / broadcaster ID の取得用）
- **目的**: CM 区間の検出と、チャンネルの永続的な識別子の取得
- **保存**: マニフェスト本文や GraphQL レスポンスは保存しません。必要な値（CM 区間情報・チャンネル ID）のみを抽出して利用します

## 収集しないデータ

- 閲覧履歴、アナリティクス、テレメトリは**収集しません**
- Twitch やその他のサイトでの閲覧ページは**追跡しません**
- 音声・映像データそのものを**記録・送信しません**
- 拡張機能の開発者はユーザーのデータを**受信・保存・アクセスしません**
- データの販売、第三者との共有、広告目的での利用は**一切行いません**

## データの送信先

ありません。この拡張機能は**外部へのネットワークリクエストを一切行いません**。HLS マニフェストや GraphQL レスポンスは Twitch のページ自身が発行する通信を読み取るだけであり、拡張機能が新たな外部送信を発生させることはありません。すべてのデータはデバイス上に留まります。

## データの保存とセキュリティ

- すべての設定は `chrome.storage.local` に保存され、この拡張機能のみがアクセス可能です
- デバイス間での同期やクラウドへの保存は行いません
- 拡張機能をアンインストールすると、ローカルに保存されたすべてのデータが削除されます

## 権限

| 権限 | 理由 |
|---|---|
| **storage** | チャンネル音量設定とユーザー設定をローカルに保存 |
| **host_permissions** (`twitch.tv`) | Twitch ページにコンテンツスクリプトを注入し、再生音声の計測と Web Audio API による音量制御を実行 |
| **host_permissions** (`ttvnw.net`) | Twitch の HLS マニフェストを読み取り、CM 区間を検出 |

## リモートコード

この拡張機能は**リモートコードを使用しません**。すべての JavaScript は拡張機能パッケージ内にローカルで同梱されています。`page-bridge.js` はページのメインワールド (`"world": "MAIN"`) で実行されますが、これは AudioContext の制御と Twitch のページデータを読み取るためのローカルコードであり、リモートから取得されたものではありません。

## 単一用途

この拡張機能の用途は1つです: ITU-R BS.1770 ラウドネス計測と Web Audio API を使用して、**Twitch のチャンネルごとの音量設定を計測・記憶・自動適用する**こと。

## 第三者依存

なし。外部ライブラリ、SDK、CDN、アナリティクスツールは一切使用していません。

## ポリシーの変更

変更がある場合は更新日を改訂の上、本ページに掲載します。変更後も拡張機能を継続使用した場合、改訂後のポリシーに同意したものとみなします。
